# 多功能转换器

一个本地运行的 Windows 桌面工具，当前支持：

- 图片提取文字
- 音频转文字
- 视频转文字
- 视频转音频

这个项目目前以源码方式运行，暂未提供正式安装包。所有处理都在本地完成，不依赖云端服务。

## 适合谁

- 想把图片内容识别成文本的人
- 想把音频或视频转成文字的人
- 想把视频单独导出音频的人
- 想批量处理多个文件，并且控制输出顺序与命名的人

## 当前功能

- 支持文件和文件夹输入
- 支持批量任务
- 支持自定义输出文件名
- 支持两种输出方式
  - 独立文件
  - 合并成一个文件
- 合并输出按当前列表顺序生成
- 支持拖拽调整输入顺序
- 历史记录可查看、可重跑
- 图片 OCR 不依赖 Whisper 模型
- 文本输出统一为 `UTF-8`

## 当前状态

- 平台：`Windows 10/11 x64`
- 运行方式：本地源码运行
- 桌面壳：`Tauri 2`
- 前端：`React + TypeScript + Vite`
- 本地处理内核：`Python`

## 使用前准备

当前仓库版需要这些环境：

- `Node.js`
- `Python 3.13`
- `Rust stable`
- `ffmpeg`

说明：

- 音频转文字、视频转文字需要本地 Whisper 模型
- 图片提取文字不需要 Whisper 模型
- 当前源码版默认依赖系统可用的 `ffmpeg`
- 如果 `ffmpeg` 不在系统 `PATH`，可以在应用设置里指定路径

## 怎么使用

### 1. 添加任务

打开应用后，点击右上角的 `添加任务`。

### 2. 选择任务类型

可选：

- 图片提取文字
- 音频转文字
- 视频转文字
- 视频转音频

### 3. 添加输入

- 可以选择单个或多个文件
- 也可以直接选择文件夹
- 文件夹会递归展开
- 输入列表支持拖拽排序

如果你选择“合并成一个文件”，最终输出顺序会严格按这个列表顺序生成。

### 4. 选择输出方式

支持两种模式：

- `独立文件`
  - 每个输入生成一个输出
  - 如果填写了自定义输出名，多文件会自动按顺序编号
- `合并成一个文件`
  - 所有输入合并成一个输出文件
  - 合并顺序按输入列表顺序

### 5. 设置输出名称

你可以自定义输出名称：

- 合并模式下：直接生成一个你指定名字的文件
- 独立模式下：会以你指定的名字作为前缀，并自动编号

### 6. 开始处理

添加完成后，任务会进入任务列表。主界面会显示：

- 任务类型
- 输入
- 输出
- 输出方式
- 当前进度
- 当前状态

## 模型说明

音频转文字和视频转文字依赖 Whisper 模型。

首次使用前，请在应用里的“模型”页面完成模型准备。默认推荐使用：

- `medium`

如果本机已经有模型目录，也可以直接指定本地模型目录。

## 常见问题

### 图片提取文字为什么不需要模型？

图片 OCR 走的是本地 `RapidOCR`，和 Whisper 是两条不同的处理链。

### 为什么 `doctor` 说找不到 ffmpeg？

当前源码版不会自动把 ffmpeg 打进仓库运行环境里，所以你需要：

- 把 `ffmpeg` 放进系统 `PATH`
- 或者在应用设置中手动指定 `ffmpeg` 路径

### 输出文件保存在哪里？

由你在任务弹窗里选择的输出目录决定。

### 设置和历史记录保存在哪里？

保存在系统应用数据目录下：

- 设置：`settings.json`
- 历史记录：`history.sqlite3`
